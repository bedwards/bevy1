# Documentation Library Index

A collection of long-form essays written from first principles. Each essay is designed to be read as flowing prose — minimal headers, no jargon without explanation, a joy to read aloud.

2. [The Entity Component System](bevy/02-entity-component-system.md) — The architectural foundation: entities, components, systems, archetypes, and why ECS replaced inheritance
3. [Systems and Scheduling](bevy/03-systems-and-scheduling.md) — How Bevy systems work, automatic parallelism, schedules, system sets, and run conditions
4. [Resources and State Management](bevy/04-resources-and-state.md) — Singleton data, game states, transitions, events vs messages
5. [Components, Queries, and Data Modeling](bevy/05-components-and-queries.md) — The art of designing components, required components, query patterns, and filters
6. [The Rendering Pipeline](bevy/06-rendering-pipeline.md) — From data to pixels: the dual-world architecture, PBR, GPU-driven rendering, ray tracing
7. [Building User Interfaces](bevy/07-user-interface.md) — Bevy's ECS-native UI, layout with Taffy, widgets, and the contrast with immediate-mode GUI
8. [The Asset System](bevy/08-asset-system.md) — Loading assets asynchronously, handles, hot-reloading, and the glTF pipeline
9. [Relationships and Hierarchy](bevy/09-relationships-and-hierarchy.md) — Entity-to-entity connections, the ChildOf/Children system, entity disabling, and cloning
10. [Observers, Events, and Messages](bevy/10-observers-and-events.md) — Communication between systems: observable events, buffered messages, and lifecycle hooks
11. [The Plugin Architecture and Ecosystem](bevy/11-plugins-and-ecosystem.md) — How plugins work, the community ecosystem, Avian physics, bevy_egui, and organizing game code
12. [Performance and Optimization](bevy/12-performance-and-optimization.md) — Cache-friendly ECS, GPU-driven rendering, profiling, and the architecture-performance connection
